# Walrus

[English README](README.md)

Walrus یک ربات تلگرام است که فایل‌ها را از تلگرام یا لینک مستقیم دانلود می‌کند و در روبیکا آپلود می‌کند. این پروژه برای جریان‌های شخصی انتقال فایل ساخته شده؛ جایی که تلگرام نقش پنل کنترل را دارد و روبیکا مقصد ذخیره یا ارسال فایل است.

## هشدار

این پروژه برای تحقیق، یادگیری و استفاده‌ی شخصی منتشر شده است.
از آن برای سوءاستفاده، اسپم، دسترسی غیرمجاز، نقض حریم خصوصی یا هر کار آسیب‌زا و غیرقانونی استفاده نکنید.
مسئولیت استفاده‌ی درست، رعایت قوانین پلتفرم‌ها، قوانین محلی و حقوق دیگران با خود شماست.

## قابلیت‌ها

- دریافت ویدیو، سند، صدا، عکس و دیگر فایل‌های تلگرام در چت خصوصی
- دریافت لینک مستقیم فایل مثل `https://...pdf`، `https://...zip` و `file:///...mp4`
- پشتیبانی از چند لینک مستقیم در یک پیام و صف کردن جداگانه‌ی هر لینک
- ارسال خلاصه‌ی نهایی بعد از پردازش یک پیام چندلینکی
- صف آپلود محلی برای جلوگیری از هم‌زمانی چند کار
- نمایش زنده‌ی پیشرفت دانلود و آپلود
- تلاش دوباره‌ی خودکار برای خطاهای موقت روبیکا
- لغو دانلود، آپلود فعال یا کارهای داخل صف
- تلاش دوباره برای انتقال‌های ناموفق، اگر فایل دانلودشده هنوز روی سرور وجود داشته باشد
- دکمه‌های سریع برای وضعیت، انتقال‌ها، پاک‌سازی، لغو و تلاش دوباره
- ارسال پیام تکمیل بعد از آپلود موفق
- نمایش زمان کل انتقال بعد از آپلود موفق
- تغییر شماره یا سشن روبیکا از داخل تلگرام
- انتخاب مقصد آپلود از داخل تلگرام
- پشتیبانی از Saved Messages روبیکا و کانال‌های اخیر روبیکا
- آپلود فایل‌ها با نسخه‌ی خوانا و امن‌شده‌ی نام اصلی فایل

## پیش‌نیازها

- Python 3.9 یا جدیدتر
- `API_ID` تلگرام
- `API_HASH` تلگرام
- توکن ربات تلگرام

## نصب

```bash
git clone https://github.com/rezaaa/walrus.git
cd walrus
cp .env.example .env
```

## تنظیمات

فایل `.env` را در ریشه‌ی پروژه ویرایش کنید:

```env
API_ID=your_telegram_api_id
API_HASH=your_telegram_api_hash
BOT_TOKEN=your_telegram_bot_token
TELEGRAM_SESSION=walrus
RUBIKA_SESSION=rubsession
OWNER_TELEGRAM_ID=123456789
```

متغیرها:

- `API_ID` - از https://my.telegram.org
- `API_HASH` - از https://my.telegram.org
- `BOT_TOKEN` - از BotFather
- `TELEGRAM_SESSION` - نام اختیاری سشن Pyrogram؛ مقدار پیش‌فرض `walrus` است
- `RUBIKA_SESSION` - نام یا مسیر سشنی که `rubpy` استفاده می‌کند
- `OWNER_TELEGRAM_ID` - اختیاری؛ اگر تنظیم شود فقط همین کاربر تلگرام می‌تواند از ربات استفاده کند

`TELEGRAM_SESSION` نام فایل سشن تلگرام/Pyrogram را کنترل می‌کند، مثل `walrus.session`.
`RUBIKA_SESSION` نام فایل سشنی را کنترل می‌کند که `rubpy` روی دیسک می‌سازد، مثل `rubsession.rp`.
تغییر حساب روبیکا از داخل تلگرام، سشن احرازهویت‌شده‌ی همان نام تنظیم‌شده را جایگزین می‌کند و نیازی به ویرایش `.env` یا ری‌استارت ربات ندارد.

برای گرفتن شناسه‌ی عددی تلگرام خود:

- یکی از پیام‌های خود را به [@userinfobot](https://t.me/userinfobot) فوروارد کنید
- یا به [@RawDataBot](https://t.me/RawDataBot) پیام بدهید و مقدار `from.id` را بردارید

سپس آن عدد را در `.env` به عنوان `OWNER_TELEGRAM_ID` قرار دهید.
اگر خالی بماند، ربات برای همه باز می‌ماند.

## راه‌اندازی حساب روبیکا

Walrus راه‌اندازی حساب روبیکا را از داخل تلگرام انجام می‌دهد.

بعد از اجرای برنامه، ربات تلگرام را باز کنید و بفرستید:

```text
/start
```

اگر هنوز سشن روبیکا ذخیره نشده باشد، ربات مرحله‌به‌مرحله راهنمایی می‌کند و فایل سشن را خودکار می‌سازد.

مراحل راه‌اندازی یا تغییر حساب:

1. برای راه‌اندازی اول `/start` را بفرستید، یا وارد `⚙️ Settings` شوید.
2. روی `📱 Change Account` بزنید یا دستور `/set_rubika` را اجرا کنید.
3. شماره‌ی روبیکا را بفرستید.
4. اگر روبیکا رمز حساب خواست، رمز را داخل ربات بفرستید.
5. منتظر درخواست کد یک‌بارمصرف بمانید.
6. کد را بفرستید.

بعد از ورود موفق، سشن فعلی روبیکا جایگزین می‌شود و worker برای آپلودهای بعدی از همان استفاده می‌کند.

در نسخه‌های فعلی `rubpy`، سشن معمولا به صورت یک فایل `.rp` ذخیره می‌شود؛ مثلا `rubsession.rp`.

## نصب روی سرور با اسکریپت

یک‌بار پکیج‌های سیستمی را نصب کنید:

```bash
apt update && apt install -y git python3 python3-venv screen
```

پروژه را کلون و `.env` را تنظیم کنید:

```bash
cd /opt
git clone https://github.com/rezaaa/walrus.git
cd /opt/walrus
cp .env.example .env
nano .env
```

اسکریپت راه‌اندازی را اجرا کنید:

```bash
bash update.sh
```

`update.sh` در صورت نیاز `venv/` می‌سازد، وابستگی‌ها را نصب می‌کند، screen قبلی با نام `walrus` را متوقف می‌کند و برنامه را در یک screen جدید اجرا می‌کند.

برای بررسی یا وصل شدن به برنامه:

```bash
screen -ls
screen -r walrus
```

سپس ربات تلگرام را باز کنید و راه‌اندازی روبیکا را با `/start` کامل کنید.
برای جدا شدن از screen بدون توقف برنامه، `Ctrl + A` و سپس `D` را بزنید.

## به‌روزرسانی

برای به‌روزرسانی‌های بعدی از همان اسکریپت استفاده کنید:

```bash
cd /opt/walrus
bash update.sh
```

اسکریپت کد را به‌روز می‌کند، وابستگی‌ها را تازه می‌کند، screen را ری‌استارت می‌کند و همان `.env` قبلی را نگه می‌دارد.

تنظیم روبیکا و مقصد آپلود از داخل رابط تلگرام انجام می‌شود. بعد از به‌روزرسانی، ربات را باز کنید و `/start` یا `/settings` را اجرا کنید تا حساب روبیکا و مقصد آپلود را بررسی کنید.

اگر نصب قدیمی بعد از آپدیت رفتار عجیب داشت، دوباره `bash update.sh` را اجرا کنید. اگر باز هم درست نشد، پروژه را از نو کلون کنید، `.env` خود را کپی کنید و راه‌اندازی روبیکا را از داخل ربات کامل کنید.

## اجرای دستی

اگر نمی‌خواهید از `screen` یا `update.sh` استفاده کنید:

```bash
cd walrus
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
python main.py
```

این دستورها دو بخش اصلی را اجرا می‌کنند:

- `telegram_bot.py` - دریافت‌کننده و دانلودکننده‌ی تلگرام
- `rubika_worker.py` - worker آپلود به روبیکا

## کنترل‌های ربات

دکمه‌های منوی اصلی:

- `📊 Status`
- `📋 Transfers`
- `🧹 Cleanup`
- `🛑 Cancel`
- `⚙️ Settings`

دستورهای موجود:

- `/start` - باز کردن منوی اصلی
- `/settings` - نمایش سشن فعلی روبیکا و مقصد آپلود
- `/set_rubika` - شروع راه‌اندازی شماره‌ی روبیکا در تلگرام
- `/set_rubika <phone_number>` - شروع مستقیم راه‌اندازی با شماره
- `/status` - نمایش دانلودهای فعال، آپلود فعال، صف، تعداد خطاها و مصرف فضای محلی
- `/transfers` - نمایش دانلودها، آپلود، صف و انتقال‌های ناموفق قابل تلاش دوباره
- `/cleanup` - پیش‌نمایش فایل‌های قابل پاک‌سازی در `downloads/`
- `/cleanup confirm` - حذف فایل‌های امن برای پاک‌سازی
- `/cancel` - نمایش دکمه‌های لغو برای کارهای فعال
- `/cancel <task_id>` - لغو یک کار مشخص
- `/retry <task_id>` - صف کردن دوباره‌ی آپلود ناموفق، اگر فایل محلی هنوز وجود داشته باشد
- `/retry_all` - صف کردن دوباره‌ی همه‌ی خطاهای قابل تلاش دوباره

دکمه‌های پیام انتقال:

- `🛑 Cancel` برای دانلود یا آپلود فعال و کارهای داخل صف
- `🔁 Retry` برای انتقال‌های ناموفق
- `🔁 Retry All Failed` در صفحه‌ی Transfers وقتی خطای قابل تلاش دوباره وجود دارد

انتقال موفق:

- پیام وضعیت اصلی با زمان کل انتقال به‌روز می‌شود
- ربات یک پیام تکمیل جداگانه ارسال می‌کند

مقصدهای آپلود:

- مقصد پیش‌فرض Saved Messages روبیکا است
- از `⚙️ Settings` وارد `📬 Destination` شوید تا مقصد را تغییر دهید
- می‌توانید `☁️ Saved Messages` یا `📣 Choose Channel` را انتخاب کنید
- برای آپلود در کانال، حساب فعال روبیکا باید اجازه‌ی ارسال در آن کانال را داشته باشد
- انتقال‌هایی که قبلا صف شده‌اند، همان مقصد زمان صف شدن را نگه می‌دارند

آپلود از لینک مستقیم:

- یک پیام شامل لینک مستقیم فایل بفرستید
- می‌توانید چند لینک مستقیم را در یک پیام بفرستید
- بعد از پایان پردازش چند لینک، ربات یک خلاصه با تعداد queued، failed و cancelled می‌فرستد
- schemeهای پشتیبانی‌شده: `https://`، `http://` و `file://`
- لینک باید به خود فایل اشاره کند، نه صفحه‌ی وب
- پسوندهای رایج ویدیو، تصویر، صدا، سند، PDF و آرشیو مثل `.pdf` و `.zip` پشتیبانی می‌شوند

## سیاست تلاش دوباره

آپلودهای روبیکا در خطاهای موقت خودکار دوباره تلاش می‌شوند.

- حداکثر تلاش: `5`
- تاخیر پایه: `3` ثانیه
- الگوی انتظار: `3s`، `6s`، `9s`، `12s`، `15s`

اگر همه‌ی تلاش‌ها شکست بخورد، کار در `queue/failed.jsonl` نوشته می‌شود و فایل دانلودشده‌ی محلی برای بررسی یا تلاش دوباره نگه داشته می‌شود.

نکته: وقتی دانلود کامل شده و فقط آپلود شکست خورده باشد، دکمه یا دستور Retry همان فایل موجود در `downloads/` را دوباره وارد صف آپلود می‌کند. فایل دوباره از تلگرام یا لینک دانلود نمی‌شود، مگر اینکه فایل محلی پاک شده باشد.

## لغو

سه راه برای لغو انتقال وجود دارد:

- زدن دکمه‌ی `🛑 Cancel` روی پیام انتقال
- اجرای `/cancel` و انتخاب کار از دکمه‌ها
- اجرای `/cancel <task_id>`

رفتار لغو:

- دانلود تلگرام: در اولین فرصت متوقف می‌شود
- صف آپلود: بلافاصله حذف می‌شود
- آپلود روبیکا: در اولین نقطه‌ی امن متوقف می‌شود

## ذخیره‌سازی

فایل‌های runtime:

- `downloads/` - فایل‌های دانلودشده‌ی موقت
- `queue/tasks.jsonl` - کارهای منتظر آپلود
- `queue/processing.json` - کاری که در حال آپلود است
- `queue/failed.jsonl` - لاگ کارهای ناموفق
- `queue/cancelled/` - نشانه‌های لغو
- `queue/settings.json` - تنظیمات سشن فعال روبیکا

رفتار پاک‌سازی:

- آپلود موفق: فایل محلی حذف می‌شود
- کار لغوشده: فایل محلی حذف می‌شود
- آپلود ناموفق: فایل محلی نگه داشته می‌شود

## رفع مشکل

اگر ربات اجرا نمی‌شود:

- `.env` را بررسی کنید
- وابستگی‌ها را نصب کنید
- مطمئن شوید اطلاعات تلگرام معتبر است

اگر آپلودها شکست می‌خورند:

- سشن روبیکا را بررسی کنید
- اگر مقصد کانال است، مطمئن شوید حساب روبیکا اجازه‌ی ارسال در کانال دارد
- `queue/failed.jsonl` را بررسی کنید
- مطمئن شوید فایل هنوز در `downloads/` وجود دارد
- اگر پردازش کشته شده، حافظه و swap سرور را بررسی کنید

## الهام

این پروژه بعد از چند تجربه‌ی ذخیره‌سازی و آپلود شکل گرفت:

- اول Arvan OSS را به عنوان مقصد امتحان کردم، اما کند بود.
- بعد Google Drive را امتحان کردم، اما فیلتر شد.
- بعد از آن [caffeinexz/Tele2Rub](https://github.com/caffeinexz/Tele2Rub) را دیدم و از آن برای امتحان کردن روبیکا الهام گرفتم.
- نام **Walrus** از سریال Black Sails گرفته شده: کشتی Captain Flint، یعنی Walrus.

جریان کلی Walrus ساده و صف‌محور است:

1. ربات تلگرام یک فایل را در چت خصوصی دریافت می‌کند، یا یک لینک مستقیم فایل را از پیام متنی می‌گیرد.
2. فایل در `downloads/` دانلود می‌شود.
3. یک کار در `queue/tasks.jsonl` اضافه می‌شود.
4. worker روبیکا فایل را آپلود می‌کند.
5. پیام وضعیت تلگرام در تمام مسیر انتقال به‌روز می‌شود.
